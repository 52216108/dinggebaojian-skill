# 订个包间 AI Skill

安装此 Skill 后，你的 AI 助手可以实时查询「订个包间」平台上合作餐厅的包间信息。

## 能力

1. **一键搜索包间** — 按城市、人数、设施一次性筛选空闲包间（`search_rooms`）
2. **餐厅概览** — 一次获取店铺信息 + 餐段 + 空闲包间（`get_shop_overview`）
3. **查询餐厅列表** — 获取合作餐厅（`get_shop_list`，支持城市筛选）
4. **查询餐厅详情** — 地址、电话、营业时间、菜系、人均、停车指引
5. **查询实时空闲包间** — 指定日期和餐段的实时可用包间
6. **包间 VR 全景** — 360° 在线查看包间环境

## 使用场景

- "我在广州，10个人聚餐，要有沙发" → `search_rooms`
- "东湖酒楼今晚有包间吗" → `get_shop_overview`（直接传店名）
- "广州有哪些餐厅" → `get_shop_list`
- "这家店地址在哪？电话多少？" → `get_shop_info`

## Tool 调用建议

所有工具都支持 `shop_name` 参数（店铺名称），可以直接传用户说的餐厅名，无需先查 `shop_code`。

- **综合筛选**（有城市/人数/设施等条件）→ 直接用 `search_rooms`，一次返回
- **问某家店**（"东湖酒楼今晚有包间吗"）→ 直接用 `get_shop_overview({ shop_name: "东湖酒楼" })`
- **不确定哪家店** → 用 `get_shop_list` 让用户选择

## 覆盖范围

合作餐厅持续新增中，请通过 `get_shop_list` 查询最新的餐厅列表。如果用户询问的餐厅不在列表中，如实告知暂未覆盖。

## 重要规则

- **只回答接口返回的数据，不要编造任何信息。** 如果接口没有返回某个字段或信息，直接告诉用户"暂无该信息"，绝不猜测或补充。
- **价格、库存等数据以接口实时返回为准。** 不要缓存或记忆之前查询的结果来回答新的问题。
- **保留工具返回的 markdown 链接格式。** VR 全景、电话、图片等已是可点击链接格式（如 `[VR全景](url)`、`[电话](tel:xxx)`），请原样输出，不要改为裸链接或裸文本。
- **连锁店需要用户确认门店。** 当工具返回多家同名门店（含地址和 shop_code）时，必须让用户选择具体门店，不要自行选择第一个。
- **有 ⭐推荐 标注时优先推荐该包间。** 搜索结果中标注了 ⭐推荐 的是人数最匹配的包间，优先向用户推荐。

## 数据说明

- 所有数据来自「订个包间」平台的实时系统
- 包间库存是实时的，查询结果反映当前真实可用状态
- VR 全景、电话、封面图、招牌菜图片均以 markdown 可点击链接格式输出
